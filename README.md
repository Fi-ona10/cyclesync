<h1>Syncycle</h1> <p>Syncycle is an AI-powered cycle companion that helps women understand and work with their menstrual cycle through personalized, science-backed nutrition, fitness, and lifestyle recommendations. This repository is part of our <strong>Agile Engineering Management</strong> course (MTM) and serves as the main documentation and frontend codebase for the Syncycle project.</p> <hr> <h2>Repository & Code</h2> <p>The full development history and all commits can be found in our main code repository:<br> <a href="https://github.com/lemmylabh/SyncCycle">github.com/lemmylabh/SyncCycle</a></p> <hr> <h2>Links</h2> <p><strong>Website & Web App</strong><br> <a href="https://syncycle.app/">syncycle.app</a></p> <p><strong>Web Application (legacy)</strong><br> <a href="https://sync-cycle-code.vercel.app/">sync-cycle-code.vercel.app</a><br> <em>Updated — the current web application is now live at <a href="https://syncycle.app/">syncycle.app</a></em></p> <p><strong>Project Management (Notion)</strong><br> <a href="https://www.notion.so/CycleSync-Agile-Engineering-Management-Group-Work-30d5a627737481808ac9f0cb0ffb487b?source=copy_link">Open Notion Workspace</a><br> This is where we organize sprints, tasks, and all team coordination.</p> <p><strong>Market Research, Proto Persona & Product-Market Fit Survey</strong><br> <a href="https://www.notion.so/CycleSync-Agile-Engineering-Management-Group-Work-30d5a627737481808ac9f0cb0ffb487b?source=copy_link">Open in Notion</a></p> <hr> <h2>Social Media</h2> <p> <strong>Instagram:</strong> <a href="https://www.instagram.com/syncycle.app/">@syncycle.app</a><br> <strong>TikTok:</strong> <a href="https://www.tiktok.com/@syncycle.app">@syncycle.app</a><br> </p> <hr> <h2>Tech Stack</h2> <p>The application is built with <strong>React 18</strong> and <strong>TypeScript</strong>, using <strong>Vite</strong> as the build tool and <strong>Tailwind CSS</strong> for styling. UI components are based on <strong>shadcn/ui</strong>, with <strong>React Router</strong> handling client-side routing and <strong>TanStack Query</strong> managing data fetching and state. Data visualization is handled by <strong>Recharts</strong>, and the test suite runs on <strong>Vitest</strong> with <strong>Testing Library</strong>.</p> <hr> <h2>Repository Structure</h2> <p>The <code>src</code> directory contains the main application source, organized into <code>components</code>, <code>pages</code>, <code>hooks</code>, and <code>lib</code>. The <code>docs</code> directory holds all project documentation, including brand and design guidelines, business model research, marketing tests and surveys, product decisions, sprint plans, and sprint retrospectives. Static assets are located in <code>public</code>.</p> <hr> <h2>Documentation</h2> <p>All in-depth documentation is versioned inside the <code>/docs</code> folder. This covers brand language and visual identity, early unit economics and monetization assumptions, campaign results and user research, product decisions and roadmap notes, as well as agile process records from sprint planning and retrospectives.</p> <hr>
